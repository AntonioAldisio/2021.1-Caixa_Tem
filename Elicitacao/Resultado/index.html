    Resultado Elicitação

    1. Metodologia

    O objetivo desse documento é reunir todos os requistos que foram elicitados para o projeto a partir das técnicas:

    Legenda

    1. IF - Introspecção Funcional
    2. BF - Brainstorming Funcional
    3. SF - Storytelling Funcional
    4. OF - Observação Funcional
    5. EF - Entrevistas Funcional
    6. QF - Questionário Funcional
    7. AF - Analise de Documentos Funcional

    Após discussão com o grupo, decidimos utilizar a técnica de priorização MoSCoW, pois a técnica é mais apropriada para o contexto do aplicativo.[1]

    2. Requisitos Funcionais

    ID Requisito Técnica Priorização
    RF 01 O usuário deve ser capaz de realizar cadastro BF01, OF1, SF01 MUST
    RF 02 O usuário deve ser capaz de realizar login IF01, SF02, BF02, OF02 MUST
    RF 03 O usuário pode realizar pagamentos SF08, BF03 SHOULD
    RF 04 O usuário pode realizar pagamentos via boletos QF06, EF06, IF04 SHOULD
    RF 05 O usuário pode realizar pagamentos via PIX IF05, OF09 COULD
    RF 06 O usuário tem que ser capaz de pagar praticamente qualquer tipo de conta nas Unidades Lotéricas IF16 SHOULD
    RF 07 O usuário pode consultar saldo QF02, EF05, IF03, SF06, BF04, OF03 MUST
    RF 08 O usuário pode consultar o extrato QF03, IF02, SF07, BF05, OF04 MUST
    RF 09 O sistema deve gerar cartão de débito virtual QF08, EF08, IF19, BF06 SHOULD
    RF 10 O usuário poderá sacar dinheiro em qualquer caixa eletrônico da caixa. EF03, SF09, OF07 WOULD
    RF 11 O sistema deve gerar código para saque QF05, IF08, BF07 COULD
    RF 12 O usuário deve ser capaz de comprar na maquininha com QR code QF07, EF07, IF09, BF08 WOULD
    RF 13 O usuário pode consultar informe de rendimentos BF09 WOULD
    RF 14 O usuário pode realizar transferências QF01, EF04, BF10, OF05, OF08 MUST
    RF 15 O usuário tem que ser capaz de realizar transferência via TED/DOC IF06 MUST
    RF 16 O usuário tem que ser capaz de realizar transferência via pix IF07 MUST
    RF 17 O usuário pode consultar comprovantes IF20, BF11 MUST
    RF 18 O usuário pode receber o seguro desemprego pelo app BF12 SHOULD
    RF 19 O usuário tem que ser capaz de validar o dispositivo IF11 COULD
    RF 20 O usuário tem que ser capaz de consultar Auxílio Emergencial IF12 MUST
    RF 21 O usuário tem que ser capaz de consultar BEm IF13 COULD
    RF 22 O usuário tem que ser capaz de consultar FGTS Emergencial IF14 COULD
    RF 23 O usuário tem que ser capaz de consultar Bolsa Família IF15 COULD
    RF 24 O usuário pode realizar recarga de celular IF17, BF13 WOULD
    RF 25 O usuário pode fazer recarga de transporte IF23, BF14 WOULD
    RF 26 O usuário tem que ser capaz de fazer recarga no aplicativo IF22 WOULD
    RF 27 O usuário tem que ser capaz de acessar as informaçōes do NIS IF21 COULD
    RF 28 O usuário tem que ser capaz de movimentar sua conta poupança social digital na CAIXA IF18 COULD
    RF 30 O usuário poderá checar se está elegível a receber auxílio emergencial. SF03, SF10 MUST
    RF 31 O usuário pode checar o estado do seu auxílio emergencial. EF02, SF04, OF06 MUST
    RF 32 O usuário deverá ter acesso ao calendário que marca o dia que o seu auxílio será disponibilizado. SF05 MUST
    RF 33 O usuário terá opção de contratar seguro apoio família. SF11 COULD
    RF 34 O usuário terá opção de Selecionar funções favoritas. EF01 WOULD

    4. Requisitos Não - Funcionais

    ID Requisito Técnica Priorização
    RNF 01 O aplicativo deve ter compatibilidade com qualquer sistema operacional BNF01 MUST
    RNF 02 O sistema deve recusar o acesso de pessoas não autorizadas INF04, BNF02 MUST
    RNF 03 As funcionalidades devem se parecer com uma conversa (chatbot) QNF03, SNF02, BNF04 COULD
    RNF 04 O aplicativo não pode ter grande espaço de armazenamento. INF01 SHOULD
    RNF 05 O aplicativo não pode ficar indisponível por mais que 10 minutos por dia. INF02, SNF03 MUST
    RNF 06 O aplicativo não pode apresentar dados de cunho privativo que não seja do próprio usuário. INF03 MUST
    RNF 07 O usuários devem conseguir encontrar a funcionalidade desejada em menos que três clicks. INF05 SHOULD
    RNF 08 O aplicativo deve se comunicar com o banco de dados. INF06 MUST
    RNF 09 O aplicativo deve proteger os dados dos usuários. QNF01, INF07, SNF05 MUST
    RNF 10 O aplicativo deve ser desenvolvido em uma linguagem que tenha uma comunidade ativa e/ou que não tenha previsão de descontinuade da linguagem nos próximos 5 anos. INF08 SOULD
    RNF 11 O usuário não pode ficar mais de 15 minutos em fila para acessar o aplicativo. SNF01 SHOULD
    RNF 12 O sistema deve ser acessível para Pessoas com Deficiência (PcD) QNF02, ENF02, BNF03 MUST
    RNF 13 O código de segurança gerado para realizar compras com cartão de débito virtual deve ser válido para apenas uma compra QNF04 MUST
    RNF 14 Ter a opção de guia para primeiro acesso ENF03, ONF01 COULD
